Overview

Web frontend development standards help teams build pages with consistency, quality, accessibility, and maintainability in mind. They also support collaboration by making projects easier to read, test, validate, and update over time.

Many of these practices align with the Front-End Checklist and MDN guidance, including semantic HTML, responsive design, valid CSS, meaningful metadata, accessible images, strong heading structure, and clean JavaScript behavior.

Checklist

Project setup

  • Use lowercase, semantic file and folder names.
  • Keep filenames clear and meaningful.
  • Verify there are no broken internal or external links.

Head requirements

  • Include doctype, charset, viewport, title, description, and author.
  • Load external CSS in the head in the correct order.
  • Use a matching title and h1 topic.

Accessibility

  • Use one h1 and ordered heading levels.
  • Provide alt text for meaningful images.
  • Check contrast, keyboard focus, and responsive behavior.

Head

Required metadata

The head of the document should include the doctype, language attribute, UTF-8 character set, viewport settings, a descriptive title, a meta description, and a meta author element.

External references

Reference external CSS files with link elements and attach JavaScript with the defer attribute so the document can parse before scripts run.

HTML

Semantic structure

Use semantic elements such as header, nav, main, section, article, aside, and footer to give the page a meaningful structure for both users and assistive technology.

Validation

Ensure the page is W3C compliant, heading levels are logical, and all links and content behave as expected.

CSS

Maintainable styling

Use external CSS, prefer classes over IDs for styling, and keep selectors readable, reusable, and appropriately scoped.

Responsive presentation

Design for readability, alignment, contrast, and consistency across screen sizes without horizontal scrolling or dead space.

JavaScript

Clean behavior

Published pages should not contain console output, unused functions, or script errors. Keep JavaScript focused on real interactions that improve usability.

Other standards

  • Optimize images to fit the design and preserve aspect ratio.
  • Make sure every non-decorative image has relevant alt text.
  • Keep page weight under 500KB when possible.
  • Review accessibility warnings and errors before publishing.
  • Check spelling, grammar, responsiveness, and visual consistency.
  • Avoid third-party CSS frameworks unless your course allows them.
